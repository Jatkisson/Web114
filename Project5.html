Questions

1. What is the difference between a function declaration and a function expression?
A function declaration is when a function is declared in a more familiar way to most other programming language, but a function expression is when a function can be declared like it is a variable. When it is declared like a variable, then it can be used in the same way as a variable is declared or it can be declared as if it is a part of an object. Function declarations are also different because they are available to be used whenever they are needed in a script, but function expressions must have the function already expressed or else it will not be able to be called in the script.
2. Why do we create functions in the first place? There are two big reasons. One has to do with WHEN the statements in a function run. The other has to do with how often the statements in a function are run.
Functions are needed because tasks can be needed at different times within a website. If a function is used, then it can execute the task when the page is loaded or it can be used to respond to an action on the page. When a function is used, then tasks can be triggered at times beyond when the page is loaded. The other benefit is that we can reuse the code in the function if it is written in a function and not listed on its own in the script tags. When functions are written, then they can be reused to simplify the code as well as improving the maintenance of the code. When task are written within functions, then you only need to modify the function once and the calls to the function will all be updated right away.
